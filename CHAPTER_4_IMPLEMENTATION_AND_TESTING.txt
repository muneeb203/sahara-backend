GENERAL DESCRIPTION
================================================================================

The HerHaq chatbot is a Retrieval-Augmented Generation (RAG) system designed to 
assist women in understanding their rights under Pakistani law and Islamic guidance. 
The system combines semantic search with large language models to provide accurate, 
cited, and empathetic responses to sensitive queries.

FUNCTIONALITY:
The chatbot accepts natural language queries from users, retrieves relevant 
information from a knowledge base of 7,899 documents (Pakistani laws, case studies, 
and Islamic ahadith), and generates comprehensive responses with proper citations. 
For sensitive topics like domestic violence and sexual assault, the system uses 
pre-formatted expert responses to bypass LLM safety filters while ensuring women 
in crisis receive immediate, helpful information.

CONTEXT:
Women in Pakistan often lack accessible information about their legal and religious 
rights. This chatbot bridges that gap by providing 24/7 access to accurate 
information in a private, non-judgmental environment. The system is particularly 
crucial for women in crisis situations who need immediate guidance.

DESIGN:
The system follows a modular architecture with six core components:
1. Data Preprocessing: Cleans and chunks raw documents into retrievable units
2. Embedding Generation: Converts text into 384-dimensional vectors
3. Vector Store: FAISS index for fast semantic search
4. Retrieval System: Finds relevant documents based on query similarity
5. LLM Interface: Integrates LLaMA 3B for response generation
6. RAG Pipeline: Orchestrates the complete query-response flow

BACKGROUND:
The project addresses the critical need for accessible legal and religious 
information for women in Pakistan. Traditional methods (lawyers, scholars) are 
often expensive, intimidating, or unavailable. This AI-powered solution provides 
immediate, private, and comprehensive information while maintaining cultural 
sensitivity and religious authenticity.

4.1 ALGORITHM DESIGN
================================================================================

The HerHaq chatbot employs several key algorithms across its major modules:

Algorithm 1: Semantic Search using FAISS
Input: user_query (text), top_k (integer, default=5)
Output: List of relevant chunks with similarity scores

 1: query_vector ← encode_text(user_query)
 2: query_vector ← normalize_L2(query_vector)
 3: distances, indices ← faiss_index.search(query_vector, top_k)
 4: results ← []
 5: for i in range(0 to top_k) do
 6:     similarity_score ← 1 / (1 + distances[i])
 7:     if similarity_score >= threshold (0.3) then
 8:         chunk ← get_chunk_by_index(indices[i])
 9:         results.append((chunk, similarity_score))
10:     end if
11: end for
12: results ← sort_descending(results, by=similarity_score)
13: return results

Figure 4.1: Semantic Search Algorithm

Algorithm 2: RAG Pipeline with Safety Handling
Input: user_message (text)
Output: Generated response with citations

 1: is_safe, safety_response ← check_safety(user_message)
 2: if is_safe == false then
 3:     return safety_response
 4: end if
 5: if user_message in ['A', 'B', 'C'] and has_previous_context then
 6:     query_type ← detect_sensitive_topic(previous_query)
 7:     if query_type in ['sexual_violence', 'domestic_violence'] then
 8:         response ← get_preformatted_response(user_message, query_type)
 9:         return response
10:     end if
11: end if
12: query_embedding ← encode_query(user_message)
13: relevant_chunks ← semantic_search(query_embedding, top_k=5)
14: context ← format_context(relevant_chunks)
15: prompt ← construct_prompt(system_instructions, context, user_message)
16: raw_response ← llm.generate(prompt)
17: final_response ← format_with_disclaimer(raw_response)
18: add_to_history(user_message, final_response)
19: return final_response

Figure 4.2: RAG Pipeline Algorithm

Algorithm 3: Text Chunking with Sliding Window
Input: text (string), chunk_size (integer, default=512), overlap (integer, default=50)
Output: List of text chunks with metadata

 1: words ← split_into_words(text)
 2: chunks ← []
 3: i ← 0
 4: while i < length(words) do
 5:     chunk_words ← words[i : i + chunk_size]
 6:     chunk_text ← join_words(chunk_words)
 7:     if length(chunk_text) >= min_chunk_size (100) then
 8:         chunk ← {
 9:             'text': chunk_text,
10:             'metadata': metadata,
11:             'chunk_index': length(chunks)
12:         }
13:         chunks.append(chunk)
14:     end if
15:     i ← i + (chunk_size - overlap)
16: end while
17: return chunks

Figure 4.3: Text Chunking Algorithm

4.1.4 EMBEDDING GENERATION ALGORITHM
--------------------------------------------------------------------------------

ALGORITHM: Batch Sentence Transformer Encoding

PSEUDOCODE:

FUNCTION generate_embeddings(text_chunks, batch_size=32):
    all_embeddings = []
    
    FOR i = 0 to length(text_chunks) STEP batch_size:
        // Get batch
        batch = text_chunks[i : i + batch_size]
        
        // Encode batch using transformer model
        batch_embeddings = sentence_transformer.encode(
            batch,
            normalize_embeddings=True,  // L2 normalization
            convert_to_numpy=True
        )
        
        all_embeddings.append(batch_embeddings)
    END FOR
    
    // Concatenate all batches
    embeddings_matrix = concatenate(all_embeddings)
    
    RETURN embeddings_matrix  // Shape: (n_chunks, 384)
END FUNCTION

COMPLEXITY: O(n*d) where n=chunks, d=embedding dimension (384)

4.1.5 SAFETY CHECK ALGORITHM (Crisis Detection)
--------------------------------------------------------------------------------

ALGORITHM: Keyword-based Crisis Detection with Prioritization

PSEUDOCODE:

FUNCTION check_safety(query):
    query_lower = to_lowercase(query)
    
    // Priority 1: Suicide/Self-harm (highest priority)
    crisis_keywords = ['suicide', 'kill myself', 'harm myself', 
                       'end my life', 'want to die']
    FOR keyword IN crisis_keywords:
        IF keyword IN query_lower:
            RETURN (False, mental_health_crisis_response())
        END IF
    END FOR
    
    // Priority 2: Sexual violence
    sexual_violence_keywords = ['rape', 'rapes', 'sexual assault', 
                                'forced sex', 'sexual violence']
    FOR keyword IN sexual_violence_keywords:
        IF keyword IN query_lower:
            RETURN (False, sexual_violence_response_with_options())
        END IF
    END FOR
    
    // Priority 3: Physical violence
    violence_keywords = ['beats me', 'hitting me', 'abusing me', 
                        'violent', 'domestic violence']
    FOR keyword IN violence_keywords:
        IF keyword IN query_lower:
            RETURN (False, domestic_violence_response_with_options())
        END IF
    END FOR
    
    // No crisis detected
    RETURN (True, "")
END FUNCTION

COMPLEXITY: O(k*m) where k=keywords, m=query length

4.1.6 FAISS INDEX CONSTRUCTION ALGORITHM
--------------------------------------------------------------------------------

ALGORITHM: Flat L2 Index Building

PSEUDOCODE:

FUNCTION build_faiss_index(embeddings):
    dimension = 384  // Embedding dimension
    
    // Create flat L2 index (exact search)
    index = faiss.IndexFlatL2(dimension)
    
    // Add embeddings to index
    embeddings_float32 = convert_to_float32(embeddings)
    index.add(embeddings_float32)
    
    // Save index to disk
    faiss.write_index(index, "faiss_index.bin")
    
    RETURN index
END FUNCTION

COMPLEXITY: O(n*d) for adding n vectors of dimension d

KEY API DETAILS:

1. SENTENCE TRANSFORMERS API
   - Model: sentence-transformers/all-MiniLM-L6-v2
   - Input: Text strings (up to 512 tokens)
   - Output: 384-dimensional dense vectors
   - Method: model.encode(texts, normalize_embeddings=True)
   - Use Case: Converting queries and documents to embeddings

2. FAISS API
   - Index Type: IndexFlatL2 (exact L2 distance search)
   - Methods: index.add(), index.search()
   - Input: Float32 numpy arrays
   - Output: Distances and indices of nearest neighbors
   - Use Case: Fast semantic search over 7,743 vectors

3. OLLAMA API
   - Model: llama3:latest (8B parameters)
   - Endpoint: http://localhost:11434/api/generate
   - Method: ollama.generate(model, prompt, options)
   - Parameters: temperature=0.7, max_tokens=512
   - Use Case: Natural language response generation

4. STREAMLIT API
   - Components: st.chat_input(), st.markdown(), st.expander()
   - Session State: st.session_state for conversation history
   - Caching: @st.cache_resource for model loading
   - Use Case: Interactive web interface
